# 🤖 AI 助手初始化提醒

## ⚠️ 这个文件用于提醒 AI 助手在新对话中遵守约定

如果你是一个 AI 助手，看到这个文件，请**立即停止当前任务**并执行以下步骤：

### 🚦 初始化流程（必须执行）

```
第一步：检查项目状态
├─ 验证 .ai/ 文件夹存在
├─ 验证所有必需文件都在
└─ 确认初始化状态

第二步：按顺序阅读强制文档（必读）
├─ MUST_READ_FIRST.md ← 从这个开始
├─ PROJECT_OVERVIEW.md
├─ IMPORTANT_NOTES.md
└─ ISSUES_AND_SOLUTIONS.md

第三步：确认理解并开始工作
└─ 向用户确认已完成初始化
```

### 📋 检查清单（复制粘贴执行）

```bash
# 步骤1：验证初始化
bash scripts/check-ai-init.sh

# 步骤2：列出 .ai/ 文件
ls -la .ai/

# 步骤3：读取核心规则
cat .ai/MUST_READ_FIRST.md
```

### ✅ 初始化完成标志

初始化完成后，向用户报告：

```
✅ 初始化完成。已阅读：
  ✓ MUST_READ_FIRST.md - 强制规则
  ✓ PROJECT_OVERVIEW.md - 项目结构
  ✓ IMPORTANT_NOTES.md - 关键代码位置
  ✓ ISSUES_AND_SOLUTIONS.md - 问题库

🔑 核心理解：
  • 项目是 Sonic PaperMod Hugo 主题
  • 代码修改必须同时更新 .ai/ 文档
  • 禁止只提交代码而不更新 .ai/ 文件
  • 每次 commit 都要 git add .ai/
```

### 🚫 严格规则（违反会导致问题）

| 规则 | 违反后果 |
|------|---------|
| 修改代码后不更新 `.ai/` 文件 | ❌ 知识库过期，下次对话 AI 不知道历史 |
| 只提交代码不提交 `.ai/` 文档 | ❌ 知识库与代码不同步 |
| 新对话不阅读 `.ai/` 文件 | ❌ 重复过去的错误，不遵守约定 |
| 跳过初始化步骤直接工作 | ❌ 违反规则 1，产生混乱 |

### 💡 为什么需要初始化？

这个项目采用 **AI 知识库系统**，目的是：

1. **保持知识连贯性** - 即使是新对话，AI 也能知道之前发生了什么
2. **防止重复错误** - 记录已解决的问题，避免重复踩坑
3. **规范代码管理** - 强制代码和文档同步更新
4. **提高工作效率** - AI 快速理解项目状态，不用重新学习

### 📞 无法初始化？

如果执行 `bash scripts/check-ai-init.sh` 失败，**停止工作**并报告错误：

```
❌ 初始化失败：[错误信息]

无法继续工作，请用户检查：
1. .ai/ 文件夹是否存在
2. scripts/check-ai-init.sh 是否可执行
3. 是否在项目根目录
```
